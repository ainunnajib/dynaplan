## Architecture
- Backend: FastAPI, 1600 tests (latest run: 1600 passed, 1 skipped, 1 xfailed, 1 xpassed), file-backed SQLite with NullPool for tests (+ optional PostgreSQL test-container path)
- Engine: Pure Python formula engine in production now includes F051 time functions, F052 lookup/cross-module functions, F053 text/conversion functions, F054 advanced aggregation/statistical functions (SUMIF/COUNTIF/AVERAGEIF, MEDIAN/STDEV/VARIANCE, PERCENTILE/LARGE/SMALL, GROWTH), and F061 calendar enhancements (`PERIODOFFSET`, context-backed retail period bounds) with matching Rust formula-module implementation (`engine/dynaplan-core/src/formula/functions.rs`) pending Rust toolchain validation
- Quotas: Workspace-level quota controls (max models, max cells/model, max dimensions/model, storage MB) with API usage reporting + enforced create/write guards
- Encryption: model-scoped at-rest cell encryption with per-model key versioning/rotation (`model_encryption_keys`), encrypted payload storage (`cell_values.value_encrypted`), and pluggable KMS wrapping providers (`local`, `aws_kms`, `vault`)
- Database infra: Alembic runtime + revision chain in `backend/alembic`, asyncpg pool controls, optional read-replica session dependency
- Execution runtime foundation: async background executor (`job_executor`), scheduler wrapper (`job_scheduler`), active job registry (`job_registry`), CloudWorks connector SDK (`app/connectors`), activated CloudWorks cron runtime + webhook notifications, and pipeline execution runtime (`app/engine/pipeline_runtime`) with `/pipeline-runs/{run_id}/execute`
- Data Hub: model-scoped staging tables with schema-on-write column definitions, connector ingestion from F063 SDK, transform pipeline operations, publish-to-module runtime, and lineage tracking (`backend/app/models|services|api/data_hub.py`, frontend `DataHubManager`)
- CLI: `cli/dynaplan_cli` Click-based Anaplan Connect tool (`import`, `export`, `run-process`, `run-pipeline`, `batch`) backed by API-key authenticated public endpoints (`/api/v1/modules/*`, `/api/v1/dimensions/*`, `/api/v1/processes/*`, `/api/v1/pipelines/*`)
- Frontend: Next.js 15 + TypeScript + Tailwind + Recharts, TS compiles clean
- Cell storage: sparse table with sorted pipe-separated dimension_key
- API keys: hashed with SHA-256, scoped (read:models, write:cells, admin, etc.)
- Import/export: CSV and Excel via openpyxl
- Actions: chained processes with step ordering, async run
- Collaboration: WebSocket presence + cursor + cell broadcasts
- RBAC: workspace roles, model permissions, dimension member access
- DCA: selective access rules + driver line items for cell-level read/write control
- SSO: SAML 2.0 + OIDC provider config
- Workflows: stages, tasks, approvals, gate checks
- ALM: dev/test/prod environments, revision tags with auto snapshots, deep structural diff/change detection, conflict-aware promotion engine with additive/replace/manual strategies, execution-time rollback restore, and promotion records
- CloudWorks: scheduled imports/exports with retry/status tracking, cron runtime activation (register/enable/disable lifecycle), connector-driven execution (`/runs/{run_id}/execute`), and completion/failure webhooks
- Pipelines: no-code data orchestration with executable step chains (source/transform/filter/map/aggregate/publish), per-step records/logging, and failure-aware run lifecycle
- SCIM: automated user/group provisioning with enterprise IdP integration
- Engine profiles: Classic and Polaris-like modes for large model optimization
- Backpressure: `pytest -m backpressure` marker added with dedicated CI gate
